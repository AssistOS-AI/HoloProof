HoloProof Installation Guide

This guide explains how to prepare the environment for HoloProof and how to bootstrap the project locally.

1. Prerequisites

Use a Unix-like environment (Linux or macOS). For Windows, use WSL for the same command flow.

  1. Install Node.js 20+ and npm.
  2. Install one or both SMT solvers: Z3 and CVC5.
  3. Ensure the parent integration dependency ../AchillesAgentLib is available.

2. Solver Setup

Install the solver binaries and verify they are visible in PATH.

Ubuntu/Debian example:

sudo apt update
sudo apt install -y z3 cvc5
z3 --version
cvc5 --version

macOS example (Homebrew):

brew install z3 cvc5
z3 --version
cvc5 --version

3. Project Bootstrap

Clone and enter the repository, then install JavaScript dependencies once package.json is available.

git clone <your-repo-url> HoloProof
cd HoloProof
npm install
Current repository status is documentation-first. If runtime files are not present yet, keep this step for the first implementation commit.

4. LLM Integration Baseline

HoloProof does not define a separate provider stack in core modules. It uses the parent AchillesAgentLib API through LLMAgent and existing provider/model routing. 

import { LLMAgent } from '../AchillesAgentLib/LLMAgents/index.mjs';

const llmAgent = new LLMAgent({ name: 'HoloProof-Encoder' });

Set provider API keys and Achilles model configuration according to the parent library configuration rules.

5. Validate the Environment

Before running the system, verify each baseline dependency in one pass:

node -v
npm -v
z3 --version
cvc5 --version
ls -la ../AchillesAgentLib
